# git-training-shivanand

## What I Learned

During this Git and GitHub training, I learned the fundamentals of version control and collaborative development workflows.

### Task 1: Repository Setup

* Created and configured a GitHub account and repository.
* Cloned a remote repository to a local machine.
* Configured Git user identity using `user.name` and `user.email`.

### Task 2: First Commits

* Created a simple web page using HTML and CSS.
* Learned how to stage files, create commits, and write meaningful commit messages.
* Pushed local commits to GitHub and verified commit history.

### Task 3: Branching Workflow

* Created and worked with feature branches.
* Added new features independently using separate branches.
* Created Pull Requests (PRs) and merged changes into the main branch.
* Synced local and remote repositories and cleaned up merged branches.

### Task 4: Merge Conflict Resolution

* Understood how merge conflicts occur when multiple branches modify the same code.
* Learned to identify conflicting changes and resolve conflicts manually.
* Completed the merge process by committing and pushing the resolved changes.

### Key Skills Gained

* Git fundamentals (clone, add, commit, push, pull)
* Branching and merging strategies
* Pull Request workflow on GitHub
* Conflict resolution techniques
* Collaborative development best practices
